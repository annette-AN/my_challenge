2.
1) 화면번호 4.2 (챗봇빌더>답변>상세)
해당 답변에 관련한 이미지를 추가 부분이 필요해보입니다.
(5번 미리보기에 있는 이미지는 다음인텐트 추천에서 버튼 대신 이미지로 추가하라는 의미로 해석됩니다.)
2) 화면번호 5.1 (챗봇빌더>의도)
"문장" 열에 각 의도 당 2행을 추가하여, 1행은 예시문장을 2행은 정규식 적용을 할 수 있었으면 좋겠습니다.
"답변"창에서 작성이 완료된 후, 작성된 답변을 기준으로 "의도"창에서 인텐트 뿌려준다면,
기존 학습된 인텐트들은 예시 문장이 존재하겠지만, 새로 추가된 인텐트들은 빈칸으로 나올 것 같습니다.
또한, 기존 인텐트들도 사용고객에 따라 정규식이 필요할 경우가 있어 보입니다.
3) 화면번호 6.1 (챗봇빌더>테스트)
테스트 페이지는 사용자가 "답변"창, "의도"창 어느 창을 들어가도 계속 테스트 할 수 있었으면 좋겠습니다. 

3.
1. 2페이지
-> 해당 입력창에 예시 문구가 있으면 사용자 입장에서 이해하기 편할듯(처음 사용하는 사람이라면 어떠한 방식으로 작성해야하는지 모를 수 있음)
-> 한 챗봇내에서도 사용되는 이메일 주소가 여러개인 경우 추가 기능이 필요해보임(메일 주소가 여러개인 경우 답변에서 어떤식으로 선택할 수 있을지도 고려할 필요가 있어보임)
2. 3페이지
-> 해당 계정과 관련된 챗봇 리스트를 가져와서, 사용자가 선택할 수 있도록하면 좋을듯
-> 해당 row에 마우스 올라가면 다른 row와 차별점을 줘서 클릭이 가능하다는걸 시각적을 표현해주면 좋을듯
-> 챗봇 중에서 만약에 생성 후 실질적으로 사용되지 않는 경우가 있다면 챗봇 '활성화/비활성화' 중 현재 어떠한 상태인지 표시해줘도 좋을듯(요건 모든 챗봇이 활성화되야하는게 필수조건이면 해당하지 않는 의견임당)
3. 5페이지
-> 미리보기에서 버튼 생성시 어떤 종류의 버튼을 만들 수 있는지 드롭다운 형식으로 사용자가 선택할 수 있도록 해주면 사용하기 편해보임
4. 8페이지
-> 테스트 화면의 경우 팝업창을 띄워서 챗봇생성화면과 분리되어 사용자가 수정하면서 바로 테스트를  할 수 있도록 하면 편리할듯
